## What This Project Does

finESS is a **local Next.js + Prisma/SQLite uncertainty workbench**. Single operator on their own machine loads a CSV, computes observed statistics, trains an empirical forecast ensemble (real Python sidecar wrapping `ace_hospital.UnifiedACEEnsemble`), and optionally consults one or more LLMs for narration or for draft probabilistic graphs. It is a **pre-production local beta** — not a hosted multi-tenant service, ships no advisory content in any regulated domain, and Path A (LLM-drafted graphs) is explicitly labelled as draft via a non-dismissible UI banner.

**Tech Stack:** TypeScript (Next.js 14.2.35 App Router), React, Prisma 6.19 / SQLite, Python 3.11 / FastAPI sidecar (ace_hospital.UnifiedACEEnsemble), OpenRouter for LLMs, Jest for tests, Docker Compose for the sidecar.
**Architecture Pattern:** Local monolith (Next.js server-side API routes) with one optional Python sidecar over HTTP; no external auth; SQLite for persistence; OpenRouter as the only outbound provider gated through a budget-capped client.

- **Q1.** Should the lint warning fix be its own commit, or rolled into A5 when NodeEditor is next touched? (Cosmetic; either is fine.)
- **Q2.** Should `SemanticConversation` allow simultaneous multiple in-flight conversations per user, or enforce one-at-a-time? Plan doesn't specify; the simpler model is one-at-a-time with explicit "start new" UI.

- **Q5.** Phase D1 introduces a richer `NodeProvenance` shape that migrates the existing 3-value `source` string. The plan calls for an idempotent migration script tested against a real `dev.db` snapshot. Need to capture a "before" snapshot of the operator's local DB before this phase lands.
